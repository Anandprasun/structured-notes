Across 13 graded cases in two test sets, the skill ended every note with an accurate fidelity check (e.g. "13/13 key facts verified against source", derived numbers marked, blank fields noted). Plain Claude did this zero times. This is the failure that bit the baseline in the broad run — it invented a "Jan" date and a made-up $10.2K damages total. A model doesn't check its own numbers unless told to; the skill makes it.

So where the skill genuinely wins

Guaranteed consistency (the floor). The skill's audit + verification fire on every run regardless of context length, rush, or phrasing. In the realistic 20-case run, the unskilled baseline mostly produced no audit and no verification at all — it only "kept up" here because each test handed it a clean, focused, obviously-flawed input. Real usage isn't that tidy.
Self-verification (the trust layer). 100% vs 0%. In high-stakes notes — clinical, legal, financial — a note you can trust without re-reading the source is the whole game.
A stable house format. Same spine, same layered structure, same place for decisions/actions/gaps every time — so a stack of notes is scannable and comparable, not 20 different shapes.
What it is not. It is not "the model couldn't do this." For an undeniable capability gap you'd need deterministic artifacts a chat can't reliably emit — a valid calendar .ics, a CSV tracker, a verification report file. That layer was scoped and deferred by choice; this report reflects the audit-plus-verification skill as it stands.
